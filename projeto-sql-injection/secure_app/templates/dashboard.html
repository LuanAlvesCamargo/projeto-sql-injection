✅ Como esta versão protege contra SQL Injection
1. Prepared Statements:
   Query: SELECT * FROM users WHERE username=%s AND password=%s
   Parâmetros: passados separadamente — nunca concatenados.

2. Validação de entrada:
   Campos verificados antes de chegar ao banco.
   Ano, quantidade e tamanho de texto validados.

3. Teste de ataque:
   Tente ' OR '1'='1 no login → será tratado como texto literal.
   O banco receberá: username = "' OR '1'='1" → nenhum usuário encontrado.
